## 1. 서비스 개요

### 1.1 한 줄 정의
인형, 피규어 등 캐릭터 굿즈 수집가를 위한 **경매 기반 중고 거래 플랫폼**.

### 1.2 문제 정의 (왜 만드는가)
| 기존 채널 | 문제점 |
|---|---|
| 번개장터 / 당근 | 고정가 거래 위주. 희소품의 "진짜 시세"를 반영 못함. 카테고리 전문성 없음 |
| 트위터(X) 덕션 | DM 기반 수기 경매. 사기 빈번, 입찰 기록 없음, 결제·배송 보호 전무 |
| 야후옥션(일본) | 언어 장벽, 배대지 필요, 수수료·배송비 부담 |
| 중고나라 | 사기 위험, 상태 등급 기준 없음 |

**핵심 인사이트**: 수집품은 정가가 무의미하고 희소성이 가격을 결정한다 → 경매가 가장 적합한 가격 결정 방식. 그런데 국내에는 수집품 특화 경매 플랫폼이 사실상 없다.

### 1.3 타겟 카테고리
- **1차 (론칭 버티컬)**: **브라이스(Blythe)** — 인형 본체(네오/미디/쁘띠), 커스텀 브라이스, 아웃핏·부속. §13 상세
- **2차 확장 (인접 인형 시장)**: 구체관절인형(BJD), 오비츠·픽코니모 등 소체, 돌피드림, 캐릭터 봉제인형·아이돌 솜인형
- **3차 확장**: 피규어(넨도로이드·스케일), 아이돌 포토카드, 트레카 등
- 확장 순서 논리: 브라이스 → BJD/소체는 유저층·커스텀 문화·부속 시장이 겹쳐 자연 전이가 쉬움. 피규어는 문화가 달라 마지막

## 2. 타겟 유저 & 페르소나

### 페르소나 A — "성덕 컬렉터" (구매자 중심)
- 20대 후반~30대 직장인, 월 굿즈 지출 10~50만원
- 절판된 한정판 피규어를 찾아 트위터 덕션·야후옥션을 뒤짐
- Pain: 사기 걱정, 상태 확인 불가, 해외 직구 번거로움

### 페르소나 B — "탈덕 정리러" (판매자 중심)
- 장르를 갈아타면서 컬렉션을 한꺼번에 정리하고 싶음
- Pain: 시세를 모름 → 헐값에 팔거나 안 팔림. 하나씩 올리기 귀찮음

### 페르소나 C — "브라이스 집사" (1차 버티컬 핵심 유저)
- 20~40대 여성 비중 높음, 인형 1~수십 체 보유, 커스텀·아웃핏에 지속 지출
- 네이버 카페 장터와 트위터에서 거래, 절판 모델은 야후옥션을 배대지로 이용
- Pain: 팩토리를 정품으로 속는 사기, 선입금 먹튀, 야후옥션 언어 장벽·배대지 비용, 커스텀 작가 분양 경쟁(선착순 광클·DM 추첨)

### 페르소나 D — "커스텀 작가" (공급자이자 인플루언서)
- 페이스업·카빙·리루팅으로 커스텀 완성품을 분양. 팔로워 수백~수천
- Pain: 트위터 분양은 가격을 올려 부르기 눈치 보임(정찰제 관행) → 실수요 대비 저평가, 입금 관리·먹튀 스트레스
- 플랫폼 가치: 경매로 정당한 시장가 + 에스크로로 입금 관리 자동화

## 3. 핵심 기능 상세

### 3.1 경매 시스템 (핵심 중의 핵심)

#### 경매 방식
| 방식 | 설명 | 용도 |
|---|---|---|
| **일반 경매** | 시작가 설정, 기간 내 최고 입찰자 낙찰 | 기본 |
| **즉시구매 병행** | 경매 + Buy Now 가격 동시 설정. 즉구가 도달 시 즉시 종료 | 빨리 팔고 싶은 판매자 |
| **비공개 최저가 (Reserve)** | 최저 낙찰가 미달 시 유찰 | 고가 희소품 보호 |
| **고정가 판매** | 경매 없이 일반 판매 (2차 기능) | 저가·대량 물품 |

#### 입찰 규칙
- **자동입찰(프록시 입찰)**: 최대 입찰가를 입력하면 시스템이 현재가+입찰단위로 자동 응찰. eBay 방식
- **입찰 단위**: 현재가 구간별 자동 결정 (예: ~1만원: 500원 / ~5만원: 1,000원 / ~20만원: 5,000원 / 20만원~: 10,000원)
- **스나이핑 방지 (Soft Close)**: 마감 5분 전 입찰 발생 시 마감 5분 자동 연장 (반복). 막판 저격 방지 → 낙찰가 상승 효과
- **입찰 취소**: 원칙적 불가. 오입력은 마감 1시간 전까지 1회 취소 허용 (기록 남김)
- **입찰 자격**: 본인인증 완료 유저만. 신규 유저는 입찰 한도 제한 (예: 첫 30일 누적 20만원)

#### 미결제(낙찰 파기) 방지 — 플랫폼 신뢰의 생명선
- 낙찰 후 **24시간 내 미결제 시 자동 취소 + 페널티** (경고 → 7일 정지 → 영구 정지)
- 미결제 시 **차순위 입찰자에게 자동 승계 제안** (24시간 내 수락 여부 선택)
- 고액 경매(예: 30만원 이상)는 **입찰 보증금** 옵션 (카드 가승인, 낙찰 실패 시 자동 해제)

### 3.2 상품 등록

- **카테고리 트리**: 대분류(피규어/인형/굿즈) → 중분류(넨도로이드/BJD/솜인형…) → 작품/IP 태그 (원피스, 산리오, 세븐틴 등)
- **IP·작품 태그 시스템**: 수집가는 "작품 기준"으로 검색함. IP 태그 팔로우 → 신규 등록 알림 (킬러 기능)
- **상태 등급 표준화** (플랫폼 공통 기준):
  | 등급 | 기준 |
  |---|---|
  | S (미개봉) | 미개봉 신품. 박스 손상 여부 별도 표기 |
  | A (개봉 미전시) | 개봉만 함, 전시 안 함, 부속 완비 |
  | B (전시품) | 전시 사용감 있음, 부속 완비 |
  | C (사용감) | 도색 벗겨짐·파손·부속 누락 등 하자 명시 필수 |
- **사진 규칙**: 최소 3장(전면/후면/박스 또는 하자 부위), **당일 날짜 손메모 인증샷 1장 필수** (도용 방지)
- **등록 템플릿**: 카테고리별 맞춤 필드 (피규어: 제조사·스케일·발매년도 / BJD: 헤드·바디 조합, 메이크업 여부 / 솜인형: 사이즈 cm, 공식/비공식)
- **금지 품목**: 정품 인증 불가 짝퉁, R18 굿즈(정책 결정 필요), 도난·분실 신고 물품

### 3.3 결제 & 에스크로 (안전거래)

```
낙찰 → 구매자 결제 (플랫폼이 대금 보관)
     → 판매자 발송 (운송장 등록)
     → 구매자 수령 확인 (또는 도착 후 3일 자동확정)
     → 판매자에게 정산 (수수료 차감)
```

- **PG**: 토스페이먼츠 or 포트원(아임포트) — 에스크로/가상계좌/카드 지원
- **분쟁 처리**: 수령 후 3일 내 "상태 불일치" 신고 가능 → 사진 증빙 → 운영자 중재 → 환불/부분환불/거절
- **정산**: 구매확정 후 D+1 판매자 계좌 이체

### 3.4 신뢰 & 안전 시스템

- **본인인증**: 휴대폰 본인인증 필수 (PASS/문자). 판매자는 계좌 실명 확인 추가
- **평판 시스템**: 거래 후 상호 평가 (별점 + 태그: "설명과 일치", "포장 꼼꼼" 등). 판매자 배지 (거래 50회+ & 평점 4.8+ → "믿을 수 있는 셀러")
- **사기 방지**: 외부 거래 유도 문구 자동 감지·차단 (오픈채팅 링크, 계좌번호 패턴), 신고 시스템, 동일 사진 재사용 감지
- **검수 서비스 (3차 기능)**: 고가품(30만원+) 대상 플랫폼 경유 검수 옵션 — 번개장터 정품검수 모델

### 3.5 실시간 & 알림

- **실시간 입찰 현황**: WebSocket으로 현재가·입찰자 수·남은 시간 실시간 갱신
- **알림 (푸시/카카오 알림톡)**:
  - 상위 입찰 발생 (내가 밀렸을 때) — 재입찰 유도, 참여율 핵심
  - 관심 경매 마감 임박 (1시간 전 / 10분 전)
  - 팔로우한 IP 태그 신규 등록
  - 낙찰 / 유찰 / 결제 요청 / 발송 / 도착

### 3.6 탐색 & 커뮤니티

- **검색**: 키워드 + 필터 (카테고리, 상태등급, 가격대, 마감임박순, 신규등록순, 입찰많은순)
- **관심(찜) & 최근 본 경매**
- **시세 데이터 (차별화 포인트)**: 낙찰가 히스토리 공개 — "넨도로이드 XXX 최근 낙찰가 평균 4.2만원". 데이터가 쌓이면 강력한 해자
- **컬렉션 자랑 피드 (2차)**: 인스타식 피드. 커뮤니티가 곧 리텐션

## 4. 화면 설계 (주요 페이지)

| 화면 | 핵심 요소 |
|---|---|
| **홈** | 마감 임박 경매 캐러셀, 인기 입찰 TOP, 팔로우 IP 신규 등록, 카테고리 숏컷 |
| **경매 상세** | 사진 갤러리, 현재가+남은시간(실시간), 입찰 내역 로그, 입찰 버튼(금액 입력+자동입찰 설정), 즉구 버튼, 판매자 프로필·평점, 상태등급·상세설명, Q&A 댓글 |
| **입찰 모달** | 현재가, 최소 입찰가 자동 표시, 최대 자동입찰가 입력, 수수료 안내, 확인 2단계 |
| **등록 플로우** | 카테고리 선택 → 템플릿 폼 → 사진 업로드(인증샷 가이드) → 경매 설정(시작가/즉구가/기간/Reserve) → 미리보기 → 등록 |
| **마이페이지** | 입찰 중 / 낙찰 / 판매 중 / 판매 완료 / 정산 내역 / 찜 / 알림 설정 |
| **주문·배송** | 결제, 운송장 등록·조회, 구매확정, 분쟁 신고 |
| **판매자 프로필** | 평점, 거래 이력, 판매 중 경매 목록, 팔로우 |
| **검색 결과** | 필터 사이드바, 카드 그리드 (썸네일+현재가+남은시간+입찰수) |
| **어드민** | 신고 처리, 분쟁 중재, 유저 제재, 금지품목 모니터링, 정산 관리, 대시보드 |

**플랫폼 우선순위**: 모바일 웹 (반응형) 먼저 → PWA 푸시 → 네이티브 앱은 PMF 확인 후. 타겟층(트위터 덕후)은 링크 공유 기반 유입이라 웹이 유리.

## 5. 데이터 모델 (ERD 개요)

```
users            (id, email, nickname, phone_verified, penalty_level, rating_avg, created_at)
seller_profiles  (user_id, bank_account_verified, trust_badge, sales_count)
categories       (id, parent_id, name, template_schema JSONB)
tags             (id, type[IP/작품/캐릭터], name)         ← 유저 팔로우 대상
items            (id, seller_id, category_id, title, description,
                  condition_grade[S/A/B/C], attributes JSONB, status)
item_images      (id, item_id, url, is_proof_shot, sort_order)
item_tags        (item_id, tag_id)
auctions         (id, item_id, start_price, buy_now_price, reserve_price,
                  bid_unit_rule, starts_at, ends_at, extended_count,
                  status[scheduled/live/ended/sold/cancelled], current_price, winner_id)
bids             (id, auction_id, bidder_id, amount, max_proxy_amount,
                  is_auto, created_at, status[active/outbid/cancelled/won])
orders           (id, auction_id, buyer_id, seller_id, amount, fee,
                  status[pending_payment/paid/shipped/delivered/confirmed/disputed/refunded],
                  payment_due_at)
payments         (id, order_id, pg_tx_id, method, escrow_status)
shipments        (id, order_id, carrier, tracking_no, shipped_at, delivered_at)
settlements      (id, order_id, seller_id, amount, fee, settled_at)
reviews          (id, order_id, reviewer_id, target_id, rating, tags[], comment)
disputes         (id, order_id, reason, evidence_urls[], resolution, resolved_by)
notifications    (id, user_id, type, payload JSONB, read_at)
watchlist        (user_id, auction_id)
tag_follows      (user_id, tag_id)
reports          (id, reporter_id, target_type, target_id, reason, status)
price_history    (tag_id/item_key, sold_price, sold_at)   ← 시세 데이터
```

**핵심 트랜잭션 주의점**
- 입찰은 동시성 충돌 지점 → `SELECT ... FOR UPDATE` 또는 Redis 분산 락으로 auction row 잠금 후 현재가 검증·갱신
- 프록시 입찰 로직: 신규 입찰 시 기존 최고 프록시와 자동 경쟁 계산
- 경매 마감: 지연 큐(예: BullMQ delayed job)로 마감 처리, soft-close 연장 시 job 재스케줄

## 6. 기술 스택 제안

| 레이어 | 선택 | 이유 |
|---|---|---|
| 프론트 | **Next.js (App Router) + TypeScript + Tailwind** | SEO(검색 유입 중요) + 반응형 + 빠른 개발 |
| 백엔드 | **NestJS (또는 Next.js API + 별도 워커)** | 경매 마감 스케줄러·입찰 동시성 등 서버 로직이 많아 분리 권장 |
| DB | **PostgreSQL** | 트랜잭션 무결성 (입찰·에스크로), JSONB 템플릿 |
| 캐시/락/큐 | **Redis + BullMQ** | 입찰 락, 경매 마감 지연 잡, 실시간 현재가 캐시 |
| 실시간 | **WebSocket (Socket.io) 또는 SSE** | 입찰 현황 실시간 갱신 |
| 파일 | **S3 호환 (Cloudflare R2)** + 이미지 리사이징 | 사진 위주 서비스, 트래픽 비용 절감 |
| 결제 | **토스페이먼츠 / 포트원** | 에스크로, 정산 API |
| 인증 | 휴대폰 본인인증 (PASS/아임포트 본인인증) + 소셜 로그인 (카카오/트위터) | 타겟층 = 트위터 유저 |
| 알림 | 웹푸시(PWA) + 카카오 알림톡 | 입찰 알림이 리텐션 핵심 |
| 배포 | Vercel(프론트) + 소규모 VPS/Fly.io/Railway(백엔드·워커) | 초기 비용 최소화 |

> MVP를 더 가볍게: **Next.js + Supabase (Postgres/Auth/Realtime/Storage)** 조합도 가능. 단, 프록시 입찰·마감 스케줄러는 Edge Function/cron으로 구현해야 해서 경매 로직이 복잡해지면 한계 → 초기 검증용으로는 충분.

## 13. 브라이스 퍼스트 전략 (론칭 버티컬 상세)

### 13.1 왜 브라이스인가

| 요인 | 내용 |
|---|---|
| **높은 객단가** | 정품 신품 20~40만원대, 절판 한정판 수십만~수백만원, 유명 작가 커스텀 100만원+, 빈티지 켄너(1972) 수백만원. 수수료 모델과 궁합이 좋음 |
| **정품 검증 니즈** | 팩토리(공장 유출/무허가 조립) 문제가 시장 최대 이슈. "팩토리를 정품으로 속여 파는 사기"가 가장 흔한 피해 → **검증·에스크로의 가치가 어떤 카테고리보다 큼** |
| **경매 문화 기확립** | 수집가들이 이미 야후옥션 재팬에서 배대지로 경매 참여 중. 경매라는 방식 자체에 익숙함 → 교육 비용 없음 |
| **활발한 부속 시장** | 인형 본체 외에 아웃핏, 가발/스칼프, 아이칩, 슈즈 등 소액·고빈도 거래가 받쳐줌 → 거래 유동성 확보 |
| **커스텀 작가 생태계** | 작가(커스터머) 브랜드가 가격을 결정. 작가 분양(release)이 사실상 경매로 진행되는 문화 → 플랫폼 공식 분양 채널 기회 |
| **좁고 깊은 커뮤니티** | 국내 네이버 카페·트위터·인스타에 집중. 채널이 좁아 초기 마케팅 타깃이 명확 |

### 13.2 브라이스 시장 구조 (현황)

- **정품 유통**: 타카라토미 × CWC 기획, Junie Moon(공식 스토어). 국내는 정식 유통이 얇아 구매대행·직구 의존
- **거래 채널**: 네이버 카페(중고 장터 게시판), 트위터/X, 인스타그램 DM, 야후옥션 재팬(배대지), eBay
- **현재 거래 방식의 문제**: 카페 장터는 선입금 기반 → 먹튀 위험. DM 거래는 기록이 없음. 야후옥션은 언어 장벽 + 배대지 수수료 + 관세
- **가격 결정 요인**: 모델(발매년도·한정 수량), 개체 상태(헤어·페이스·아이 메커니즘), 커스텀 여부와 작가 네임밸류, 풀셋(박스·증지·스탠드·기본 아웃핏) 여부

### 13.3 브라이스 특화 카테고리·등록 템플릿

```
브라이스
├── 인형 본체
│   ├── 네오 브라이스 (약 28cm)
│   ├── 미디 브라이스 (약 20cm)
│   ├── 쁘띠 브라이스 (약 11cm)
│   └── 빈티지 (켄너 1972 등)
├── 커스텀 브라이스 (작가 커스텀 완성품)
├── 커스텀 소재·부속
│   ├── 아이칩 / 아이 메커니즘 부품
│   ├── 스칼프·가발
│   ├── 페이스플레이트 / 돔
│   └── 바디 (오비츠, 퓨어니모 등 교체 바디)
├── 아웃핏 (의상)
│   ├── 작가제 (핸드메이드)
│   └── 기성품 (CWC 정품 아웃핏 등)
└── 슈즈·소품 (신발, 가방, 스탠드, 미니어처)
```

**본체 등록 템플릿 필수 필드**
- 모델명 + 발매년도 (자동완성: 역대 발매 모델 DB를 구축해 선택식으로 → 데이터 품질 확보, 시세 히스토리의 기반)
- **정품 구분**: 정품 / 팩토리 / 모름 — 필수 선택, 허위 기재 시 제재 (§13.4)
- 풀셋 여부: 박스, 증지, 스탠드, 기본 아웃핏·슈즈 각각 체크
- 커스텀 여부: 디폴트(무커스텀) / 부분 커스텀 / 풀 커스텀 + **작가명**, 커스텀 내역(페이스업, 카빙, 아이칩 교체, 리루팅, 바디 교체 등 체크리스트)
- 상태 상세: 헤어 상태, 페이스 기스·변색, 아이 메커니즘(줄 당김·아이칩 전환) 작동 여부, 바디 관절 상태
- **인증샷 규칙(브라이스 특화)**: 당일 손메모 + ① 얼굴 정면 ② 뒷통수(스탬프 각인 부위) ③ 아이 메커니즘 줄 ④ 박스·증지(풀셋 주장 시) — 정품 판별 포인트를 사진 규칙에 내장

### 13.4 팩토리 브라이스 정책 (1순위 정책 결정)

시장 최대 논쟁 지점. 선택지는 셋:

| 옵션 | 장점 | 단점 |
|---|---|---|
| A. 전면 금지 | "정품 전용" 프리미엄 포지셔닝, 지재권 리스크 최소 | 실거래량의 상당 부분 포기, 유동성 손해 |
| **B. 허용하되 강제 구분 (권장)** | 시장 현실 반영 + "속여 팔기"만 정확히 타격. 팩토리 카테고리 분리·검색 필터 제공 | 지재권 이슈 잔존(무허가 조립품), 정책 커뮤니케이션 필요 |
| C. 자율 (구분 없음) | 운영 부담 없음 | 기존 채널과 차별점 소멸. 선택지에서 제외 |

**권장 B 상세**: 등록 시 정품/팩토리 필수 선택 → 팩토리는 배지·필터로 명확 구분 → **팩토리를 정품으로 판매 적발 시 즉시 영구 정지 + 전액 환불**. "이 플랫폼에서 정품 배지는 믿을 수 있다"가 핵심 셀링포인트가 됨. 단, 지재권 리스크(무허가 복제품 중개 책임)는 법률 검토 필요 — 검토 결과에 따라 A로 회귀 가능성 열어둘 것.

### 13.5 브라이스 특화 기능 (백로그)

- **모델 도감 + 시세**: 역대 발매 모델 DB(모델명·발매년도·정가·이미지) → 모델별 낙찰가 히스토리 차트. "이 모델 최근 시세" — 검색 유입(SEO)과 해자의 핵심
- **작가 프로필**: 커스텀 작가 인증 계정, 포트폴리오, 작가 팔로우 → **공식 분양 경매** (현재 트위터 선착순/DM 추첨으로 진행되는 분양을 플랫폼 경매로) — 작가에게는 정당한 가격, 플랫폼에는 킬러 콘텐츠
- **정품 검수 서비스**: 고가 거래 시 플랫폼 경유 검수(스탬프·증지·메커니즘 확인). 브라이스는 검수 가치가 특히 높아 Phase 2로 앞당길 후보
- **안전 포장 가이드**: 인형 배송 파손(특히 페이스, 아이 메커니즘) 분쟁 예방 — 발송 체크리스트 제공

### 13.6 콜드 스타트 플랜 (브라이스 GTM)

1. **작가 5~10명 선영입**: 커스텀 작가에게 분양 경매 채널 제안 (수수료 0% + 노출 지원). 작가 분양 1건 = 팔로워 유입 수백 명
2. **네이버 카페·트위터 커뮤니티 공략**: 기존 장터의 pain(선입금 먹튀)을 정면으로 — "입찰 기록 공개 + 에스크로" 메시지
3. **초기 물량 시딩**: 운영자가 직접 인기 모델 몇 개를 경매 등록해 "돌아가는 시장"처럼 보이게. 첫 낙찰 사례를 콘텐츠화
4. **트위터 공유 최적화**: 경매 링크 OG 카드에 인형 사진 + 현재가 + 마감 카운트다운 — 트위터 RT가 곧 마케팅
5. **시세 콘텐츠 SEO**: "브라이스 ○○ 모델 시세" 검색 유입 → 모델 도감 페이지가 랜딩

### 13.7 MVP 범위 조정 (브라이스 반영)

Phase 1에 **추가**: 브라이스 카테고리 트리 + 본체 등록 템플릿(정품 구분 포함) + 모델명 자동완성(초기엔 수동 구축한 모델 DB ~수백 건)
Phase 2로 **승격**: 작가 프로필 + 분양 경매 (GTM의 핵심이므로 조기 투입), 정품 검수 검토
변경 없음: 경매 엔진, 에스크로, 페널티 시스템은 카테고리 무관 공통
